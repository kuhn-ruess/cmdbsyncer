# Version 4.1

## Unreleased

### Hosts
- FIX: Bulk "Set Template" no longer wipes the existing labels of the selected hosts — only the CMDB-template assignment changes, imported labels stay intact
- NEW: Host and Object list search box accepts boolean expressions: `AND`, `OR`, `NOT` (or `!`) and parentheses, plus optional `field:value` pairs (e.g. `(prod OR stage) AND NOT test`, `hostname:web*`, `env:prod`). Bare terms match the hostname (substring), the label / inventory keys (exact — use `*` for a prefix match) and the label / inventory values (substring), so `NOT basti_test` returns hosts without a label literally named `basti_test`. Quote values to include spaces or literal `*`
- NEW: Host list has a one-click "Export CSV" button next to the saved-search bar; the export now honours the currently active filters and search expression instead of always dumping every host
- FIX: Invalid search expressions (unbalanced parentheses, dangling operators, bad regex) now flash an error message instead of raising a server error

### Checkmk
- FIX: Checkmk site updates no longer fail the pre-update downtime with an "unknown url type" error when no explicit API server address is configured — the site's own server address and name are used as the API target by default

### MySQL / MSSQL / ODBC
- FIX: SQL import no longer rejects column or table names containing umlauts and other non-ASCII letters (e.g. `EigentümerFirmaName`); identifiers are still validated against whitespace, quotes and statement separators
- FIX: The `table` field of an ODBC / MySQL account may again carry an appended `WHERE …` filter (the pre-3.12.4 behaviour); the trailing SQL is still audited for semicolons, SQL comments and destructive keywords so it cannot smuggle DROP / TRUNCATE / UPDATE / INSERT into the executed statement
- FIX: `sql_inventorize` no longer fails the second-pass "collected_by_key" write with "inventory key must be a non-empty string"; the enumeration index is stringified before the MongoDB-key validator runs

## Version 4.1.3

### Hosts
- FIX: Dots (`.`) in label and inventory key names are no longer rejected — they are valid MongoDB field names and common in real labels (FQDNs); only `$`-prefixed and empty keys are still refused
- FIX: Hosts that were auto-archived (not seen on import / stale auto-archive) are reactivated automatically when they reappear in the source on a later import; manually deleted hosts stay archived

## Version 4.1.2

### Hosts
- FIX: Label names containing `.` or `$` are again run through the configured `REPLACERS` before the MongoDB-key check, so imports that relied on replacers to sanitize such names no longer fail

### Rules
- FIX: "Export all rules" now also includes cronjob groups, so a full rule export/import no longer silently drops the schedule configuration

### UI
- FIX: Settings menu entry "Approvals" renamed to "Critical Label Approvals" and only shown when `APPROVAL_REQUIRED_LABELS` is actually configured (was always visible in CMDB mode even when the workflow was inactive)

## Version 4.1.1

### API
- FIX: Separate `API_RATE_LIMIT` for `/api/v1` (default `30/min; 300/hour`) so a few 401s no longer lock out monitoring polling for the rest of the hour
