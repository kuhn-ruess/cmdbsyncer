# Version 4.1

## Unreleased

### Ansible
- FEAT: Trigger and audit Ansible playbook runs from the UI — pick a host, run a playbook, see live output and the recorded run history per host
- FEAT: Modernize the default Ansible roles for current OS versions
- FEAT: Wire the Playbook Fire Rule outcome to the discovery dropdown so admins pick from cataloged playbooks instead of typing names by hand
- FEAT: Catalog playbooks via a manifest with friendly names; a `.local` override lets a deployment add or rename entries without forking the manifest
- FEAT: Preview button runs `ansible-playbook --check --diff` against the selected host so the operator sees what would change before triggering a real run
- FEAT: Cross-module inventory provider registry — any plugin can register an inventory source, and the Ansible inventory endpoint lists every registered provider
- FEAT: Ansible Projects — pick a rule source per project; each project is served as its own inventory provider so different teams can run against scoped subsets of hosts
- FEAT: Auto-default project + project-centric workspace — opening the Ansible UI picks a sensible default project and the views are organised around it
- FEAT: CLI moved under `cmdbsyncer ansible …` with a run-time provider picker; backward-compat shims keep the old top-level commands working
- FIX: Inventory URL now resolves at `/api/v1/ansible/inventory/<provider>` (was the legacy path)

### Hosts
- FIX: Bulk label edit and bulk template assignment fetch the selected
  hosts in a single `id__in` query instead of one round-trip per host —
  per-doc `.save()` stays so HostLabelChange signals still fire
- FEAT: Bulk label edit and bulk template assignment redirect back to
  the list page the user was on (with their pagination, sort and
  filters intact) instead of dumping them on the unfiltered first page.
  Cancel buttons follow the same return path. The redirect target is
  validated as a relative path so it can't be turned into an open
  redirect

### Refactor
- REFACTOR: `application/views/host.py` extracted ~370 lines —
  WTForms widgets/fields moved to `host_widgets.py`, Flask-Admin Mongo
  filters moved to `host_filters.py`. Public symbols (`HostModelView`,
  `ObjectModelView`, `TemplateModelView`) unchanged
- REFACTOR: Second wave of host.py split — column formatters / Markup
  renderers (~660 lines) moved to `host_renderers.py`, the
  `HostnameAndLabelSearchMixin` joined the Mongo filters in
  `host_filters.py`. `host.py` is now 1553 lines, down from 2619

### API
- FIX: `/api/v1/objects/all` rejects `limit > 10000` with HTTP 400 so a
  single client can no longer ask the server to materialise the entire
  host collection in one response

### Plugin runtime
- FIX: HTTP `requests.Session` and CA-cert temp files are now released
  via a dedicated `atexit` cleanup hook registered from `Plugin.__init__`,
  so a subclass init that raises before `_init_complete` flips to True
  no longer leaks an open Session for the lifetime of the process

### Misc
- FIX: `/api/v1/rules/<type>` now logs (with a 200-char preview) instead
  of silently swallowing rule documents whose JSON is corrupt
- FIX: Plugin discovery walks `application.plugins` + `plugins/` once and
  caches the dotted module list, instead of running `pkgutil.iter_modules`
  a second time when registering admin views

### Checkmk
- FEAT: Checkmk Rule Management list view rebuilt — long ruleset
  names and value templates wrap inside the rendered card instead of
  pushing the table off-screen (column hard-capped at 720px), the
  listing defaults to sort + visual cluster by `primary_ruleset`
  (gradient header rows announce each ruleset group with a per-group
  rule count), and a new "Ruleset contains" filter with a `<datalist>`
  autocomplete picks from the rulesets currently in use. The toolbar
  quick-search now also matches the rule's primary ruleset and any
  outcome's ruleset, not just the rule name. `primary_ruleset` is
  denormalised on save and lazily backfilled for pre-existing rows
  via an atomic `update_one` (no `name`-required validation hit on
  partial fetches)
- FIX: A Checkmk Rule Mngmt rule may carry several outcomes, but all
  outcomes have to share the same `ruleset` value. Saving with mixed
  rulesets is now rejected with a flash message — mixing rulesets in
  one rule breaks the group-by-ruleset listing and is almost always
  a copy-paste mistake. The denormalised `primary_ruleset` field is
  hidden from the column list and the edit form — the operator only
  ever sees the ruleset on the outcomes themselves

### UI
- FEAT: `/request-password` got a dedicated template with the same
  branded card layout as the login screen — gradient background, logo,
  inline icons on the input and a "Back to login" link, replacing the
  generic `wtf.quick_form` formular
- FEAT: Password-reset email rebuilt as a proper HTML message —
  branded gradient header, prominent CTA button, plain-text fallback
  link, "didn't ask for this?" notice and explicit 60-minute validity.
  Greeting falls back to the user's email when `name` is empty (the
  template previously used a non-existent `firstname` attribute and
  rendered an empty greeting)

### Auth
- SEC: Per-CronGroup webhook trigger tokens are now stored as
  SHA-256 hashes (`CronGroup.webhook_token_hash`); the plaintext is
  shown to the operator exactly once via a flash after generation /
  rotation and never persisted again. The hash field is hidden from
  the admin form — the form carries a "Regenerate webhook token on
  save" checkbox instead. Pre-4.1 plaintext rows auto-upgrade on the
  next webhook trigger so no manual migration step is required.
  Compatible with the enterprise `webhook_signatures` HMAC flow,
  which still uses its own plaintext `signing_secret` (HMAC needs
  the symmetric key)
- FEAT: New `tests/test_api_csrf_invariant.py` walks every Resource
  under `application/api/*` with `ast` and asserts each HTTP-verb method
  carries `@require_token` (or `@require_api_role` / `webhook_auth`)
  somewhere in its decorator stack or body. Locks in the precondition
  for the `csrf.exempt(api)` call in `application/__init__.py` so a
  reviewer can't silently add a session-auth route under `/api/v1` and
  bypass CSRF protection
- FIX: API auth now emits a `user.login.collision` audit event when a
  login string matches more than one User row (the historical
  duplicate-name case). Iteration over all candidates is constant-time —
  a wrong password against the first colliding user takes the same time
  as the last, so timing cannot leak which duplicate exists. Branch is
  intentionally kept (purging duplicates would lock out users whose
  `name` was empty when the field was introduced)

### Auth & Email
- FIX: Password-reset tokens now use `time.time()` for `iat`/`exp` — the
  previous mix of `datetime.now()` (signing) and `datetime.utcnow().timestamp()`
  (validation) on non-UTC hosts let tokens stay valid past their nominal
  expiry by the host's UTC offset
- FIX: `/request-password` does an equivalent JWT-signing decoy when the
  email is unknown so response time can no longer be used to enumerate
  registered addresses
- FIX: `send_email` no longer spawns a daemon thread per call; SMTP errors
  now surface in the syncer Log instead of being silently dropped
- FIX: Replace bare `except:` in `send_email` with `except RuntimeError:` —
  the unrelated `KeyError` from a missing `ext_app` kwarg now actually fails
  loudly
