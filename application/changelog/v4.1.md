# Version 4.1

## Unreleased

### Ansible
- FEAT: Trigger and audit Ansible playbook runs from the UI — pick a host, run a playbook, see live output and the recorded run history per host
- FEAT: Modernize the default Ansible roles for current OS versions
- FEAT: Wire the Playbook Fire Rule outcome to the discovery dropdown so admins pick from cataloged playbooks instead of typing names by hand
- FEAT: Catalog playbooks via a manifest with friendly names; a `.local` override lets a deployment add or rename entries without forking the manifest
- FEAT: Preview button runs `ansible-playbook --check --diff` against the selected host so the operator sees what would change before triggering a real run
- FEAT: Cross-module inventory provider registry — any plugin can register an inventory source, and the Ansible inventory endpoint lists every registered provider
- FEAT: Ansible Projects — pick a rule source per project; each project is served as its own inventory provider so different teams can run against scoped subsets of hosts
- FEAT: Auto-default project + project-centric workspace — opening the Ansible UI picks a sensible default project and the views are organised around it
- FEAT: CLI moved under `cmdbsyncer ansible …` with a run-time provider picker; backward-compat shims keep the old top-level commands working
- FIX: Inventory URL now resolves at `/api/v1/ansible/inventory/<provider>` (was the legacy path)
